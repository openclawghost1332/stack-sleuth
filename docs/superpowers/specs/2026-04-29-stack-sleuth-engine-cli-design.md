# Design: Stack Sleuth engine + CLI

Generated by brainstorming on 2026-04-29
Status: APPROVED
Mode: Builder

## Problem Statement
Stack Sleuth already turns pasted traces into a likely culprit and a checklist, but it is trapped inside the browser demo. The next meaningful step is to turn the repo into a reusable stack-trace triage tool that works in both browser and terminal workflows without splitting the logic.

## What Makes This Cool
The same deterministic engine can power a phone-friendly webpage, a terminal command, and copy-ready incident summaries. That gives the repo real product thickness while keeping the wow factor obvious in 30 seconds: paste a messy trace, get a culprit, confidence, signature, and practical next steps.

## Premises
1. The strongest path this cycle is depth, not another new toy repo.
2. Deterministic parsing and diagnosis are more trustworthy for this product than vague AI suggestions.
3. A shared engine plus CLI is enough to make Stack Sleuth feel like a real tool, not just a demo.

## Approaches Considered

### Approach A: Thin CLI wrapper
- Summary: Keep parse and diagnose separate, add a small CLI that stitches them together.
- Effort: S
- Risk: Low
- Pros: Fastest ship, smallest diff, easy to explain.
- Cons: Logic remains fragmented, browser and CLI formatting drift risk stays high, harder to extend.
- Reuses: Existing `src/parse.js`, `src/diagnose.js`.

### Approach B: Shared analysis engine plus CLI
- Summary: Add one engine entry point that returns normalized analysis data, then use it from both CLI and browser.
- Effort: M
- Risk: Low-Med
- Pros: Better architecture, keeps browser and CLI aligned, creates a reusable core for future integrations.
- Cons: Slightly more work this cycle, requires README and UI refresh.
- Reuses: Existing parser, diagnosis patterns, browser app structure.

### Approach C: Full incident platform push
- Summary: Add uploads, history, report sharing, and more languages.
- Effort: XL
- Risk: High
- Pros: Biggest long-term surface area.
- Cons: Bloats the cycle, adds storage/product complexity, likely misses a clean public ship.
- Reuses: Very little beyond current parser.

## Recommended Approach
Choose Approach B because it adds real product depth with a clean architecture and still fits a single aggressive cycle.

## Architecture
- Keep `src/parse.js` focused on runtime detection and frame extraction.
- Keep `src/diagnose.js` focused on deterministic pattern mapping.
- Add `src/analyze.js` as the shared engine that combines parse output, diagnosis output, stable signature generation, support-frame selection, and report rendering helpers.
- Add `bin/stack-sleuth.js` as the terminal entry point for stdin/file input and text/json/markdown output.
- Update `src/main.js` and `index.html` to surface the new shared-engine fields in the browser.

## Data Flow
1. Input enters from browser textarea, stdin, or a file path.
2. `analyzeTrace()` normalizes empty-input handling and calls `parseTrace()`.
3. The parsed report flows into `diagnoseTrace()`.
4. The engine attaches a stable signature plus support frames and exposes render-ready report strings.
5. Browser and CLI format the same analysis object for their respective surfaces.

## Error Handling
- Empty or whitespace-only input returns a user-facing empty-state in browser and a non-zero CLI failure with a clear message.
- Unreadable file path returns a non-zero CLI failure with the original filesystem error message.
- Unknown trace shapes still return a generic medium-confidence diagnosis instead of crashing.

## Testing
- Add engine tests for signatures, supporting-frame selection, and markdown/text render helpers.
- Add CLI tests for stdin, file input, JSON mode, Markdown mode, and empty-input failure.
- Keep existing parse and diagnose tests green.
- Extend README contract tests to cover CLI usage.

## Success Criteria
- `npm test` passes.
- Browser demo shows signature and supporting frames using the shared engine.
- CLI can summarize a trace from stdin or a file path in at least text and JSON modes.
- README clearly presents Stack Sleuth as browser + terminal tool.
